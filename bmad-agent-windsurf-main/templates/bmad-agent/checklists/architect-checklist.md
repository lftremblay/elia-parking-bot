# Architect Checklist

Use this checklist when creating or reviewing architectural designs.

## Problem Definition
- [ ] Confirm problem statement and business requirements
- [ ] Identify constraints (technical, budget, timeline)
- [ ] Define success criteria and KPIs

## Architecture Design
- [ ] Define target architecture (logical + physical)
- [ ] Select core components and their responsibilities
- [ ] Define interfaces and contracts between components
- [ ] Document data flow and storage strategy

## Non-Functional Requirements
- [ ] Define security requirements and access controls
- [ ] Plan for scalability and performance
- [ ] Define observability strategy (logging, metrics, tracing)
- [ ] Plan for disaster recovery and backup

## Operations & Maintenance
- [ ] Document deployment topology and strategy
- [ ] Define monitoring and alerting
- [ ] Plan for updates and maintenance windows
- [ ] Document runbooks for common operations

## Review & Validation
- [ ] Document tradeoffs and assumptions
- [ ] Review with stakeholders
- [ ] Validate against requirements
- [ ] Create or update architecture diagram
